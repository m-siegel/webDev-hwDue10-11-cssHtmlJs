Useful HTML Tags

Also provide a list of the 10 most useful tags (e.g. div) for you in HTML. I'm not really sure if you meant html tags, specifically header tags, or tags that go in the head.

Some HTML Tags

  1. html -- this one should be obvious
  2. head -- also pretty important
  3. body -- yeah, this is important, too
  4. link -- if you don't want inline CSS, use this
  5. meta -- great for SEO and accessibility
  6. title -- ditto
  7. h1, h2, etc -- have you ever tried to read lots of text without headers?
  8. nav -- use descriptive tags like nav to indicate that the element holds navigation links
  9. p -- without this, there'd be no five paragraph essay
  10. a -- for anchoring the link into the text
  11. there are a bunch more really useful ones, including: button, figure, figcaption, input, ul/ol and li, img, and picture

Header Tags, Specifically

I'm not going to list these. They're super useful, both for SEO and for humans because they structure the content. They can help search engines and people figure out what's on your page.

Head Tags, Or Tags That Go in the Head

I think this might be what was meant. These tags are critical for the browser, for SEO, and I think for assitive technologies like screen readers.

  • meta charset -- tells the browser how to display your characters.
  • title -- gives (short) desription of page that helps with SEO.
  • meta description -- provides info for SEO and may be used for a snippet on the results page.
  • meta viewport -- tells the browser how to render the page for mobile.

There are a number of other tags that can limit how your page is indexed in a search engine, whether your page can be translated, and other things, but the most important tags are listed above.

Demo Font Size Types

CSS. Create an itemized list with the same text and 6 different font sizes, use inline styling. Choose a combination of absolute and relative sizesLinks to an external site.. Explain on each item when would you use that unit by means of an example

  1. Same Text -- Why use: font normally comes in pt, so you might use this for consistency
  2. Same Text -- Why use: your text is displayed with pixels, so it's a good unit for absolute size, espeically if you don't want your font to get too small to read.
  3. Same Text -- Why use: if what matters most is how this size compares to the parent's font, use this, eg for a list within a list might be smaller than the outer list
  4. Same Text -- Why use: similar to the previous, but for the root element, eg if you had a bunch of elements and you wanted their sizes to scale together, relative to the root.
  5. Same Text -- Why use: this scales with the width of the screen. if you want your text to grow or shrink with the window, you could use this
  6. Same Text -- Why use: maybe you're talking about a specific real-world size
